Setup guide

Install Bake-off

Put three AI models head to head on your own computer and see which one is the most accurate, the fastest, and the cheapest to run. About 10 minutes, all free, no accounts. If you can copy and paste, you can do this.

1Install Bake-off

Open a terminal and paste this one line, then press Enter:

curl -fsSL https://raw.githubusercontent.com/jdbarzy/bake-off/HEAD/get.sh | bash

It downloads Bake-off to your home folder and sets it up as an always-on app that survives reboots. When it finishes it prints the address to open - something like http://192.168.1.50:15600/. Write that down; that address works from your phone, laptop, anything on your network.

2Get some models

Bake-off measures models; it does not include them. On each machine with an NVIDIA GPU, run the provisioner from the Bake-off folder:

bash ~/bake-off/mwboot.sh

It installs vLLM, picks a model that fits each GPU, serves it over the standard OpenAI /v1 API with a swap agent beside it, and prints the exact settings to paste into the app. The first model download is a few GB, so give it a few minutes.

Why vLLM? It is the same serving engine used in production GPU clusters, so what you measure here is what you would ship. And with the swap agent in place, the app's model catalog can change what each GPU serves with one tap.

3Open the app and run

Open the address from step 1 in any browser. From here it is three clicks, and the screen guides each one:

  • Power on - press the glowing power button. All three models should turn green ("ready").
  • Load task - pick a ready-made question card, like General knowledge.
  • Press Play - watch the three models race through the questions, then read the scoreboard and the recommendation.

That is the whole loop. Load a different card, or build your own questions, and run it again.

＋Got a setup file?

If someone already configured a Bake-off and exported its setup, you can skip configuring anything: open the app, press the gear on the control bar, go to Move setup, and import their file. Their model endpoints, comparison cards, and question sets all appear here.

?If something does not work

SymptomFix
A model shows not reachableThe model server is not running on that machine. Re-run bash ~/bake-off/mwboot.sh there, then press the re-check arrows in the app.
A model shows "not on this server"Open the model's dots menu and pick one from the list the server actually offers, or use "Switch this server to another model" - the swap agent loads it for you.
The address does not load from another deviceBoth devices must be on the same network. If it still fails, a firewall is blocking port 15600: sudo ufw allow 15600 on the Bake-off machine.
Answers are very slowCheck the model fits the GPU - a model too big for its card falls back to CPU offload. Pick a smaller one from the model catalog.
Want it gone~/bake-off/uninstall.sh removes the app and its services. Delete the ~/bake-off folder after, if you like. Downloaded models live in ~/.cache/huggingface on each GPU machine.
Privacy note: everything runs on your machines. Questions, answers, and results never leave your network unless you share them.
